<task type="auto">
  <name>Task 2: Patch EmailVerificationBanner for Supabase User compatibility</name>
  <files>src/components/EmailVerificationBanner.tsx</files>
  <action>
1. Change the import from:
   `import { useAuth } from '../hooks/useAuth';`
   to:
   `import { useAuth } from '../contexts/SupabaseAuthContext';`

2. Remove Firebase imports:
   - Delete `import { sendEmailVerification } from 'firebase/auth';`
   - Delete `import { auth } from '../firebase';`

3. Add Supabase import:
   `import { supabase } from '../lib/supabase';`

4. Change the guard check from:
   `if (!user || user.emailVerified) { return null; }`
   to:
   `if (!user || user.email_confirmed_at) { return null; }`

   Explanation: Firebase `User.emailVerified` is a boolean (true = verified). Supabase `User.email_confirmed_at` is a string timestamp or null. If non-null, the email is confirmed — so the banner should hide.

5. Change the resend call from:
   `await sendEmailVerification(auth.currentUser!);`
   to:
   ```typescript
   const { error } = await supabase.auth.resend({ type: 'signup', email: user.email! });
   if (error) throw error;
   ```

6. Keep all JSX and styling unchanged.
  </action>

4. File compiles without TypeScript errors
  </verify>

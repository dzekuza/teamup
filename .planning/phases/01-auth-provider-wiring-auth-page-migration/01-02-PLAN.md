- `import { auth } from '../firebase';` (or any firebase import)
- `import { signInWithEmailAndPassword, signInWithPopup, GoogleAuthProvider, FacebookAuthProvider } from 'firebase/auth';`

**Logic changes:**

1. **Get auth functions from context:**
   `const { login, signInWithGoogle } = useAuth();`

2. **Replace `handleEmailLogin`:**
   - Remove `signInWithEmailAndPassword(auth, email, password)` call
   - Replace with `await login(email, password);` then `navigate('/');`
   - Wrap in try/catch, set `error` to `err.message ?? 'An error occurred during login. Please try again.'`
   - Delete the `getFirebaseErrorMessage` helper function entirely

3. **Replace `handleGoogleLogin`:**
   - Remove `signInWithPopup(auth, new GoogleAuthProvider())` call
   - Replace with `await signInWithGoogle();`
   - Do NOT call `navigate('/')` after signInWithGoogle — it triggers a browser redirect, so code after it won't execute
   - Only set error in the catch block, and only call `setIsLoading(false)` in catch (not finally)

4. **Delete `handleFacebookLogin` function entirely**

5. **Delete the Facebook SDK `useEffect`:**
   - The useEffect that initializes `window.fbAsyncInit` and loads the Facebook SDK script — delete entirely

6. **Delete all plaintext password cookie logic:**

   - Remove `setCookie('savedPassword', password)` writes
   - Remove `removeCookie('savedPassword')` calls
   - Remove `rememberMe` state variable and the "Remember me" checkbox if it only controlled password cookies
   - **Keep** `savedEmail` cookie logic if it exists (saving email is fine, saving password is the security bug)
   - Add a cleanup call on mount: `removeCookie('savedPassword')` to clear any legacy cookies from existing users. Place this in a useEffect that runs once.

7. **Delete Facebook OAuth button** from the JSX (both step 1 and step 2 if it appears in both)

8. **Error handling:** Replace the `getFirebaseErrorMessage` switch statement (which maps Firebase error codes to messages) with direct Supabase error message display: `err.message ?? 'An error occurred during login. Please try again.'`

**Preserve:**
- The 2-step email entry flow (step 1: enter email, step 2: enter password)

  <files>src/components/ResetPasswordDialog.tsx</files>
  <action>
Read `src/components/ResetPasswordDialog.tsx` and make the following targeted changes:

1. **Remove Firebase imports:**
   - Delete `import { auth } from '../firebase';`
   - Delete `import { sendPasswordResetEmail } from 'firebase/auth';`

2. **Add Supabase import:**
   - Add `import { supabase } from '../lib/supabase';`

3. **Replace the password reset call in `handleSubmit`:**
   - Replace `await sendPasswordResetEmail(auth, emailInput);` with:
   ```typescript
   const { error } = await supabase.auth.resetPasswordForEmail(emailInput, {
     redirectTo: `${window.location.origin}/reset-password`,
   });
   if (error) throw error;
   ```

4. **Simplify error handling:**
   - Replace the Firebase error code switch (`error.code` cases for `auth/invalid-email`, `auth/user-not-found`) with:
   ```typescript
   setError(error.message ?? 'An error occurred. Please try again later');
   ```
   - Note: Supabase's `resetPasswordForEmail` silently succeeds even for non-existent emails (security best practice), so the `user-not-found` case is no longer needed.

5. **Keep everything else:** The component's props interface, JSX, styling, success state rendering — all unchanged.

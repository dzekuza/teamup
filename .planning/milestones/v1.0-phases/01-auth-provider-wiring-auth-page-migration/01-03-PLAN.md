Read `src/pages/Register.tsx` (784 lines) and rewrite the auth logic. Preserve the existing multi-step UI flow and all styling/layout. The changes are to the auth and data logic, not the UI.

**Imports to REMOVE:**
- `import { auth } from '../firebase';` (or `import { auth, db } from '../firebase';`)
- `import { createUserWithEmailAndPassword, updateProfile, signInWithPopup, GoogleAuthProvider, FacebookAuthProvider } from 'firebase/auth';`
- `import { doc, setDoc } from 'firebase/firestore';`
- `import { db } from '../firebase';` (if separate import)
- `import { sendWelcomeEmail } from '../services/sendGridService';` (or similar — welcome email removed for Phase 1)
- Remove the `declare global` block for `window.FB` / `window.fbAsyncInit` if present

**Imports to ADD:**
- `import { useAuth } from '../contexts/SupabaseAuthContext';`

**Imports to KEEP:**
- React, useState, useEffect, useNavigate, Link
- UI component imports (MUI, icons, images, avatars)
- Keep `useCookieContext` only if used for non-password cookies

**Logic changes:**

1. **Get auth functions from context:**
   `const { register, signInWithGoogle } = useAuth();`

2. **Replace `handleSubmit` (email/password registration):**
   - Remove `createUserWithEmailAndPassword(auth, email, password)` call
   - Remove `updateProfile(userCredential.user, { displayName, photoURL })` call
   - Remove the Firestore `setDoc(doc(db, 'users', uid), {...})` call that writes user profile + sports to Firestore
   - Remove `sendWelcomeEmail(email, displayName)` call
   - Replace with: `await register(email, password, displayName);` then `navigate('/');`
   - The Supabase `register()` function in `useSupabaseAuth.ts` handles signUp + profile creation via PostgreSQL trigger. No manual Firestore write needed.
   - **Sports selection:** The multi-step flow collects `selectedSports` but the Supabase `register()` does not persist sports (no `sports` column in `profiles` table). This is acceptable — keep the sports selection UX step intact (it's good UX), just don't write the data anywhere. Remove the Firestore write that included sports.
   - Wrap in try/catch, set error to `err.message ?? 'An error occurred during registration. Please try again.'`

3. **Replace `handleGoogleRegister` (or similar name):**
   - Remove `signInWithPopup(auth, new GoogleAuthProvider())` call
   - Remove the Firestore `setDoc` that creates a profile document after Google signup
   - Replace with `await signInWithGoogle();`
   - Do NOT call `navigate('/')` after signInWithGoogle — it triggers a browser redirect
   - Only set error and setIsLoading(false) in catch (not finally)

4. **Delete `handleFacebookRegister` function entirely**

5. **Delete the Facebook SDK `useEffect`:**
   - The useEffect that initializes `window.fbAsyncInit` and loads the Facebook SDK script — delete entirely

6. **Delete all plaintext password cookie logic:**
   - Remove `setCookie('savedPassword', password)` writes
   - Remove any `getCookie('savedPassword')` reads
   - Remove `rememberMe` state if it only controlled password cookies
   - Add a cleanup `removeCookie('savedPassword')` on mount if desired (same pattern as Login)

7. **Delete Facebook OAuth button** from the JSX

8. **Error handling:** Replace any Firebase error code switch with direct Supabase error message: `err.message ?? 'An error occurred during registration. Please try again.'`

**Preserve:**
- The multi-step registration flow (name/email/password steps, sports selection step, avatar selection step — however many steps exist)
- The Google OAuth button (change handler to use Supabase)
- All Tailwind/CSS styling and layout
- Avatar selection logic (keep the UI step — avatar selection is UX, not auth)
- Sports selection UI step (keep visible, just don't write to Firestore)
- Terms of service / privacy checkbox if present
- The `navigate('/')` after successful email/password registration

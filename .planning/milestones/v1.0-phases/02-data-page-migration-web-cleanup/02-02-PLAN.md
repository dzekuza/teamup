**Delete the entire `convertTimestampsToStrings` function** (lines 32-68 approximately). Supabase returns ISO strings — this function is unnecessary.

**Rewrite the `fetchEventsForLocation` function** inside the useEffect (around line 116):
```typescript
const fetchEventsForLocation = async () => {
  if (!selectedLocation) {
    setLocationEvents([]);
    return;
  }

  setIsLoading(true);
  try {
    const { data: eventsData, error } = await supabase
      .from('events')
      .select('*')
      .eq('location', selectedLocation.name)
      .order('date', { ascending: true })
      .order('time', { ascending: true });

    if (error) throw error;

    const events = (eventsData ?? []).map(row => toAppEvent(row, []));
    setLocationEvents(events);
    setShowEvents(events.length > 0);
  } catch (error) {
    console.error('Error fetching events for location:', error);
  } finally {
    setIsLoading(false);
  }
};
```

**Rewrite the `handleEventUpdated` function** (around line 194) with the same Supabase pattern:
```typescript
const handleEventUpdated = () => {
  if (selectedLocation) {
    const fetchEvents = async () => {
      try {
        const { data: eventsData, error } = await supabase
          .from('events')
          .select('*')
          .eq('location', selectedLocation.name)
          .order('date', { ascending: true })
          .order('time', { ascending: true });

        if (error) throw error;

        const events = (eventsData ?? []).map(row => toAppEvent(row, []));
        setLocationEvents(events);
      } catch (error) {
        console.error('Error refreshing events:', error);
      }
    };
    fetchEvents();
  }
};
```

The sorting logic is now handled by Supabase `.order()` clauses — the manual `events.sort()` is no longer needed.

Note: `toAppEvent` passes `[]` for players because the Locations page does not display player lists. EventCard will get empty players arrays.
